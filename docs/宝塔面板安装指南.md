# 宝塔面板安装 IDC 分销平台指南

> 本文档指导您在宝塔面板（Linux）上一键部署 IDC 分销平台。

---

## 环境要求

| 组件 | 版本要求 | 宝塔安装方式 |
|------|---------|-------------|
| 操作系统 | CentOS 7+ / Ubuntu 20.04+ | 宝塔面板支持 |
| PostgreSQL | 16+ | 宝塔软件商店 → PostgreSQL 管理器 |
| Redis | 7+ | 宝塔软件商店 → Redis |
| Nginx | 1.24+ | 宝塔默认安装 |
| Node.js | 22+ | 宝塔软件商店 → Node.js 版本管理器 |
| CMake | 3.20+ | 需命令行安装 |
| g++ | 12+ | 需命令行安装 |

---

## 第一步：安装宝塔环境

### 1.1 安装 PostgreSQL

宝塔面板 → 软件商店 → 搜索 "PostgreSQL" → 安装 PostgreSQL 16+

安装后创建数据库和用户：
```bash
# 进入 PostgreSQL 管理器 → 添加数据库
# 或命令行执行：
sudo -u postgres psql -c "CREATE USER idcapp WITH PASSWORD 'idcapp123';"
sudo -u postgres psql -c "CREATE DATABASE idc_platform OWNER idcapp;"
```

### 1.2 安装 Redis

宝塔面板 → 软件商店 → 搜索 "Redis" → 安装 Redis 7+

安装后启动即可（无需密码，默认端口 6379）。

### 1.3 安装 Node.js

宝塔面板 → 软件商店 → 搜索 "Node.js 版本管理器" → 安装 Node.js 22+

### 1.4 安装编译工具

通过宝塔终端执行：
```bash
# Ubuntu
apt-get update && apt-get install -y cmake g++ libpq-dev libyaml-cpp-dev

# CentOS
yum install -y cmake3 gcc-c++ postgresql-devel yaml-cpp-devel
```

## 第二步：上传源码

### 方式一：从 GitHub 克隆（推荐）

宝塔面板 → 文件 → 进入 `/www/wwwroot/` → 终端执行：
```bash
cd /www/wwwroot
git clone https://github.com/Moon-Spirit/idc-platform.git
cd idc-platform
```

### 方式二：上传 ZIP

从 GitHub 下载 ZIP，在宝塔面板中上传并解压到 `/www/wwwroot/idc-platform`。

---

## 第三步：编译后端

宝塔终端执行：
```bash
cd /www/wwwroot/idc-platform/backend
mkdir -p build && cd build

# 编译（首次编译需下载Drogon依赖，约5-10分钟）
cmake .. -DBUILD_POSTGRESQL=ON -DBUILD_REDIS=ON
make -j$(nproc)
```

编译成功后，会在 `build/` 目录生成 `idc-platform` 可执行文件（约 5.8MB）。

---

## 第四步：执行数据库迁移

```bash
cd /www/wwwroot/idc-platform

# 逐个执行迁移文件
for f in backend/migrations/0*.sql; do
  sudo -u postgres psql -d idc_platform -f "$f"
done

# 授权表权限给 idcapp 用户
sudo -u postgres psql -d idc_platform -c \
  "GRANT SELECT, INSERT, UPDATE, DELETE ON ALL TABLES IN SCHEMA public TO idcapp;"
sudo -u postgres psql -d idc_platform -c \
  "GRANT USAGE, SELECT ON ALL SEQUENCES IN SCHEMA public TO idcapp;"
```

## 第五步：构建前端

```bash
cd /www/wwwroot/idc-platform/frontend
npm install
npm run build
```

构建完成后，前端文件在 `frontend/dist/` 目录。

## 第六步：配置 Nginx（宝塔网站设置）

### 6.1 添加网站

宝塔面板 → 网站 → 添加站点：
- **域名**: 你的域名（如 `idc.example.com`）或 IP
- **根目录**: `/www/wwwroot/idc-platform/frontend/dist`
- **PHP版本**: 纯静态（不需要 PHP）

### 6.2 设置反向代理

添加站点后 → 设置 → 反向代理 → 添加反向代理：

```
代理名称: idc-api
目标 URL: http://127.0.0.1:8080
发送域名: $host
```

### 6.3 配置 WebSocket

在 Nginx 配置文件中添加（宝塔 → 网站 → 设置 → 配置文件）：

```nginx
# 在 server 块中添加以下 location
location /ws/ {
    proxy_pass http://127.0.0.1:8080;
    proxy_http_version 1.1;
    proxy_set_header Upgrade $http_upgrade;
    proxy_set_header Connection "upgrade";
    proxy_set_header Host $host;
    proxy_set_header X-Real-IP $remote_addr;
    proxy_read_timeout 86400s;
}
```

### 6.4 完整 Nginx 配置参考

```nginx
server {
    listen 80;
    server_name idc.example.com;
    
    # 前端静态文件
    root /www/wwwroot/idc-platform/frontend/dist;
    index index.html;
    
    # SPA 路由回退（重要！）
    location / {
        try_files $uri $uri/ /index.html;
    }
    
    # API 反代到 C++ 后端
    location /api/ {
        proxy_pass http://127.0.0.1:8080;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    }
    
    # WebSocket 代理
    location /ws/ {
        proxy_pass http://127.0.0.1:8080;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";
        proxy_set_header Host $host;
        proxy_read_timeout 86400s;
    }
    
    # 静态资源缓存
    location /assets/ {
        expires 1y;
        add_header Cache-Control "public, immutable";
    }
    
    # 禁止访问隐藏文件
    location ~ /\. {
        deny all;
    }
}
```

## 第七步：启动后端服务

### 7.1 使用 Supervisor 进程守护（推荐）

宝塔面板 → 软件商店 → 搜索 "Supervisor管理器" → 安装

宝塔面板 → Supervisor管理器 → 添加守护进程：

```
名称: idc-platform
启动用户: root
运行目录: /www/wwwroot/idc-platform/backend
启动命令: 
  export LD_LIBRARY_PATH=/www/wwwroot/idc-platform/backend/deps/lib
  /www/wwwroot/idc-platform/backend/build/idc-platform
进程数量: 1
```

### 7.2 手动启动（测试用）

```bash
cd /www/wwwroot/idc-platform/backend
export LD_LIBRARY_PATH=$PWD/deps/lib
IDC_DB_PASSWORD=idcapp123 nohup ./build/idc-platform > /tmp/idc.log 2>&1 &
```

### 7.3 验证服务是否运行

```bash
# 检查进程
ps aux | grep idc-platform

# 测试 API
curl http://127.0.0.1:8080/api/v1/auth/login \
  -X POST -H "Content-Type: application/json" \
  -d '{"username":"admin","password":"admin123"}'

# 应该返回包含 token 的 JSON
```

宝塔面板 → 网站 → 设置 → SSL → 申请 Let's Encrypt 免费证书：

## 常见问题

### Q: 后端启动报错 "PostgreSQL is not supported"
**原因**: 编译时未安装 libpq-dev。
**解决**: 安装后重新编译：
```bash
apt-get install -y libpq-dev
cd /www/wwwroot/idc-platform/backend/build
cmake .. -DBUILD_POSTGRESQL=ON
make -j$(nproc)
```

### Q: 后端启动报错 "Redis is not supported"
**原因**: 编译时未正确链接 hiredis。
**解决**: 确保 hiredis 库在 deps 目录中：
```bash
ls /www/wwwroot/idc-platform/backend/deps/lib/libhiredis.so
```

### Q: Nginx 访问显示 404
**原因**: SPA 路由未正确配置。
**解决**: 确认 Nginx 配置中有 `try_files $uri $uri/ /index.html;`

### Q: API 返回 500 Internal Server Error
**原因**: 数据库连接失败或 SQL 错误。
**解决**: 
```bash
# 检查数据库连接
psql -h localhost -U idcapp -d idc_platform -c "SELECT 1"
# 查看后端日志
cat /www/wwwroot/idc-platform/backend/logs/idc.*.log
```

### Q: 如何修改后端配置？
编辑 `/www/wwwroot/idc-platform/backend/config.json`，或用环境变量覆盖：
```bash
IDC_DB_PASSWORD=你的密码
IDC_JWT_SECRET=你的密钥
```

## 日常运维

### 查看日志
```bash
# 后端日志
tail -f /www/wwwroot/idc-platform/backend/logs/idc.*.log

# Nginx 日志
tail -f /www/wwwroot/idc-platform/logs/nginx/access.log
```

### 重启后端
```bash
# Supervisor 管理
宝塔面板 → Supervisor → 重启

# 或命令
kill $(pgrep idc-platform)
export LD_LIBRARY_PATH=/www/wwwroot/idc-platform/backend/deps/lib
cd /www/wwwroot/idc-platform/backend && ./build/idc-platform &
```

```bash
cd /www/wwwroot/idc-platform
git pull
cd backend/build && make -j$(nproc)
cd ../../frontend && npm run build
# 重启后端
kill $(pgrep idc-platform)
cd ../backend && ./build/idc-platform &
```
